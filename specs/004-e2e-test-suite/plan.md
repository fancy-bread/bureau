# Implementation Plan: Bureau E2E Test Suite

**Branch**: `004-e2e-test-suite` | **Date**: 2026-04-19 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/004-e2e-test-suite/spec.md`

## Summary

Add a real E2E test suite (`tests/e2e/`) that drives bureau against a local clone of `fancy-bread/bureau-test` with live Anthropic API calls and `gh` CLI. Simultaneously add `python-dotenv` loading at bureau's CLI entrypoint so developers can store `ANTHROPIC_API_KEY` in `~/.bureau/.env` rather than polluting their shell profile — keeping bureau's API billing separate from Claude Code's Pro subscription auth. CI uses a GitHub Actions secret injected as an env var; no `.env` file is written in CI.

## Technical Context

**Language/Version**: Python 3.12+
**Primary Dependencies**: `python-dotenv>=1.0` (new runtime), `pytest-timeout>=2.3` (new dev dep), `subprocess` + `os` (stdlib), `gh` CLI (external, pre-installed on ubuntu-latest)
**Storage**: `~/.bureau/.env` (user-managed, never version-controlled)
**Testing**: pytest for unit tests of dotenv loading; real API + gh for E2E tests
**Target Platform**: Linux/macOS; CI on ubuntu-latest
**Project Type**: CLI extension + test module
**Performance Goals**: Each E2E test ≤ 600s
**Constraints**: No mocks in `tests/e2e/`; dotenv load is silent on missing file; shell env always wins over file
