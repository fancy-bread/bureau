# Implementation Plan: deepagents Builder Integration, Reviewer Rename, and Skills Vendoring

**Branch**: `007-deepagents-reviewer-skills` | **Date**: 2026-04-21 | **Spec**: [spec.md](spec.md)

## Summary

Replace bureau's hand-rolled Builder inner loop (50-turn Anthropic SDK agentic loop, FILE_TOOLS, SHELL_TOOLS) with `create_deep_agent` from the `deepagents` library, bridged to bureau's `BuildAttempt` / `RunState` via a ~30-line adapter. Rename the Critic persona and node to Reviewer throughout the codebase (behaviour unchanged). Vendor ASDLC SKILL.md files into `bureau/skills/default/`, organised by agent role so the Builder receives `build/test/ship` skills and the Reviewer receives the `review` skill.

## Technical Context

**Language/Version**: Python 3.14 (existing)
**Primary Dependencies**:
- `deepagents>=0.5.3` (new — `create_deep_agent`, `SkillsMiddleware`, `FilesystemMiddleware`, `MemoryMiddleware`, `SummarizationMiddleware`)
- `langgraph>=0.2` (existing — outer StateGraph, SqliteSaver checkpoint)
- `langchain-anthropic` or compatible model adapter (required by deepagents `create_deep_agent` when using Anthropic models)
- `anthropic>=0.25` (existing — Reviewer still uses SDK directly)
- `pydantic>=2` (existing)
- `typer>=0.12` (existing)

**Storage**: SQLite per-run checkpoint via `SqliteSaver` (existing, unchanged)
**Testing**: pytest (existing)
**Target Platform**: macOS / Linux (same as existing bureau)
**Project Type**: CLI tool
**Performance Goals**: End-to-end run time within ±20% of current implementation
**Constraints**: deepagents `AgentState` is orthogonal to bureau's `RunState`; a bridge adapter must translate between them without schema changes to `RunState`
