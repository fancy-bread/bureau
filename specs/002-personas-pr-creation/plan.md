# Implementation Plan: Bureau Personas and PR Creation

**Branch**: `002-personas-pr-creation` | **Date**: 2026-04-18 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/002-personas-pr-creation/spec.md`

## Summary

Replace the four stub nodes (`planner`, `builder`, `critic`, `pr_create`) with real Claude-powered persona implementations. The Builder and Critic together form the Ralph Loop: Builder iterates until tests pass (inner loop, max 3 attempts per round); Critic audits against spec and constitution and issues `pass`, `revise`, or `escalate` (outer loop, max 3 rounds). All personas use the Anthropic API with tool use and prompt caching. PR creation calls `gh pr create` via subprocess.

## Technical Context

**Language/Version**: Python 3.14
**Primary Dependencies**: langgraph 0.2+, langgraph-checkpoint-sqlite, anthropic>=0.25, typer>=0.12, pydantic>=2 (all existing)
**Storage**: SQLite per-run checkpoint (existing); memory.json per-run scratchpad (existing — extended for task plan and build attempt history)
**Testing**: pytest (existing); integration tests added per persona
**Target Platform**: macOS / Linux CLI
**Project Type**: CLI runtime
**Performance Goals**: Full pipeline completion in under 10 minutes for typical single-feature specs
**Constraints**: Anthropic API calls must use prompt caching for repeated spec/plan content; Builder subprocess commands timeout at 300s
**Scale/Scope**: Single-run, single-repo; no concurrency within a run

**Structure Decision**: Extend the existing flat package layout. Persona logic is extracted into `bureau/personas/` (prompt construction, response parsing) separate from `bureau/nodes/` (LangGraph node wiring, state management). Tool definitions live in `bureau/tools/` for reuse across Builder and Planner. No new top-level project — this is additive to the existing `bureau/` package.
